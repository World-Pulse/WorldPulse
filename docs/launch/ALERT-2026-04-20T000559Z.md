## Likely causes (from prior incidents in memory)

1. **Signals schema drift** — `signals.source_ids` is `text[]` in prod; any join against `sources.id` without `::text` cast → 500. Recent deploys could have reintroduced this. Check `project_signal_detail_500.md`.
2. **Env var not re-read** — `docker compose restart` does NOT re-read env vars; a restart without `--force-recreate` after changing `.env.prod` will leave the container with stale config.
3. **Scraper / DB chain** — if `wp_scraper` or DB auth (`wp_user`/`worldpulse_db`) is degraded, public signal queries will 500.
4. **TypeScript runtime errors** — API ships with `pnpm build || true` and 100+ TS errors; a runtime crash would surface as 500s.

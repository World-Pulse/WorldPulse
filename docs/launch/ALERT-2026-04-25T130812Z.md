Postgres error code **42703** = undefined column.
The public-signals route is selecting `source_url` from the `signals` table, but that column does not exist in the prod schema (`signals.source_ids` is the real column — `text[]` per memory `project_signal_detail_500.md`).

## Pattern

## Suspected fix

Either:
1. The route handler should select an existing column (e.g., reconstruct `source_url` by joining `signals.source_ids` against `sources.id::text` and picking a primary), OR
2. A `source_url` column needs to be added/back-filled on `signals`.

Option 1 is the safer hot-fix — schema change on launch week is risky and we already know `source_ids` is `text[]` in prod.

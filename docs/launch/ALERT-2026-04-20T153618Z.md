# ALERT — API Down — 2026-04-20 11:35 local (15:36 UTC)

**Status:** CRITICAL — Launch day, public API returning 500.

## Failing endpoint

`GET https://api.world-pulse.io/api/v1/public/signals?limit=1`

- HTTP: 500
- Response time: ~0.28s (fast fail, not a timeout)
- Web frontend (https://world-pulse.io): OK (200)
- GitHub stars: 2

## Root cause (from response body)

```
{
  "statusCode": 500,
  "code": "42703",
  "error": "Internal Server Error",
  "message": "select \"id\", \"title\", \"category\", \"severity\", \"reliability_score\", \"location_name\", created_at as published_at, \"source_url\" from \"signals\" where \"status\" = $1 order by \"created_at\" desc limit $2 - column \"source_url\" does not exist"
}
```

Postgres error code **42703** = undefined column. The public `/signals` handler is selecting `source_url` from `signals`, but that column does not exist on the prod table. This is schema drift between application code and the live database.

## Why this is critical on launch day

- Public API is the primary integration surface advertised in the launch (world-pulse.io shows "API access" as a feature).
- Every previous health pulse today (and yesterday evening) has logged `API: FAIL` — this is **not a new incident, it is an ongoing outage** that has been running for ~18+ hours across log entries.
- The homepage loads fine so casual visitors may not notice, but anyone hitting `/api/v1/public/signals` — including our own frontend's live-signal surfaces if they proxy this route — is getting 500s.

## Likely fix paths (for on-call to pick from)

1. **Fastest:** patch the query to drop `source_url` from the SELECT list, or alias from the actual column name. The `signals` table's URL field is almost certainly stored under `source_ids` (text[]) joined to `sources.url`, per `reference_api_slop_routes.md` and `project_signal_detail_500.md` memory. The handler may have been written against a stale schema.
2. **Alt:** add `source_url` as a column (migration) — only if product intent is a denormalized copy. Risky without review.
3. **Rollback:** redeploy the last known-good API image. Check `docker-compose.prod.yml` and previous tag on the server.

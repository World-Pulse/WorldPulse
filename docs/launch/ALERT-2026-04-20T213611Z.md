## Root cause (unchanged — see ALERT-2026-04-20T150632Z.md for full detail)

Postgres error 42703: `column "source_url" does not exist` on `signals` table. API handler references a column that was dropped / never migrated. Schema drift, same pattern as `project_signals_schema_drift.md` (resolved Apr 15) and `project_api_typescript_debt.md` (API ships with TS errors via `pnpm build || true`).

## Why this keeps alerting

The scheduled health pulse runs every ~30 min and is designed to alert on any API failure. This has now failed continuously since 2026-04-19T21:25Z (~24 hours). Each run creates a new alert file per the task spec.

**Next time a human reviews this:** if the standing incident is ack'd and being worked, consider either (a) silencing the alert creation while the incident is open, or (b) teaching the skill to recognize "same 500, same root cause" and dedupe into a single rolling alert.

## Recommended fix (unchanged)

Same sequence as ALERT-2026-04-20T150632Z.md:

1. SSH prod → `psql $DATABASE_URL -c "\d signals"` — confirm `source_url` column reality
2. Grep `api/` for `source_url` — find offending handler
3. Either (a) fix the SELECT to drop `source_url` / join `sources.url` via `source_ids[]`, or (b) migrate to add the column
4. Deploy with `deploy-bg.ps1` + `--force-recreate`; wait for healthy before next deploy
5. Smoke test: `curl https://api.world-pulse.io/api/v1/public/signals?limit=1` should return 200

# ALERT — API DOWN

**Timestamp (UTC):** 2026-04-24T00:19:30Z
**Local time:** 20:19 (2026-04-23)
**Status:** API FAIL / Web OK / GitHub OK (3 stars)

## Failure

- `GET https://api.world-pulse.io/api/v1/public/signals?limit=1` → **HTTP 502 Bad Gateway**
- No response body / signal count unavailable
- Web (`https://world-pulse.io`) returns 200 structurally but front-end shows "Loading live headlines…" and "Connecting to signal feed…" with no populated data — consistent with the API being down behind nginx

## Context

- Today is stated as launch day (Apr 20, 2026) per the scheduled task prompt, but wall-clock is Apr 23 evening — this task has been re-firing for 3+ days.
- The health log shows API: FAIL for every run from **2026-04-20 onward**. This is not a flap — it is a sustained outage of the public API.
- GitHub stars static at 3 across the entire outage window.

## Likely causes (ordered by prior memory)

1. **nginx upstream** pointing at `api` instead of `wp_api` (ref: `project_nginx_upstreams.md`). 502 from the edge with front-end "connecting" loop is the exact failure signature.
2. **wp_api crash-loop** — "Container wp_api Started" is not the same as healthy (ref: `feedback_container_started_vs_healthy.md`). Need `docker inspect` + `docker logs wp_api` to confirm.
3. **wp_scraper** down would cause *empty* signals, not 502 — rule out for this specific symptom.
4. **DB env-var regression** — `||` vs `??` (ref: `feedback_nullish_vs_or_env.md`); a missing/empty `DATABASE_URL` would throw at API boot.

## Recommended manual triage (cannot run from autopilot)

```
ssh prod
cd /opt/worldpulse
docker compose -f docker-compose.prod.yml ps
docker inspect --format '{{.State.Health.Status}} {{.State.Status}}' wp_api
docker logs --tail=200 wp_api
curl -s -o /dev/null -w '%{http_code}\n' http://localhost:3001/api/v1/public/signals?limit=1
grep -n 'upstream' nginx.conf
```

If nginx upstream is wrong: fix to `wp_api` and `docker compose -f docker-compose.prod.yml up -d --force-recreate nginx`.
If wp_api crash-loop: check logs for the thrown error, then `up -d --force-recreate api` after fix (restart alone won't re-read env — ref: `feedback_restart_vs_recreate.md`).

## Severity

**P0** for launch posture. Public API has been unreachable for the entire launch window. Every "launch day" health check since Apr 20 has fired this alert with no visible remediation.

## Likely causes to investigate

1. `wp_api` container crash-loop (remember: "Started" ≠ healthy — verify with `docker inspect` + `curl` + `logs`).
2. Nginx upstream misconfig — should point to `wp_api` (container name), not `api` (compose service).
3. DB connectivity from the API container (prod is `wp_user` / `worldpulse_db`, not `worldpulse`).
4. TypeScript build shipped with `pnpm build || true` — a runtime error in a previously-masked code path.
5. Stripe / JWT_SECRET / env-var rotation from today's pre-public rotation may have left the container with stale secrets; `docker compose restart` does NOT re-read env — needs `up -d --force-recreate`.

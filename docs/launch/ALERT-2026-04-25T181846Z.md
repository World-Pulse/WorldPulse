## Error body

```json
{
  "statusCode": 500,
  "code": "42703",
  "error": "Internal Server Error",
  "message": "select \"id\", \"title\", \"category\", \"severity\", \"reliability_score\", \"location_name\", created_at as published_at, \"source_url\" from \"signals\" where \"status\" = $1 order by \"created_at\" desc limit $2 - column \"source_url\" does not exist"
}
```

## Diagnosis
Schema drift between API code and prod DB. The query in the `/api/v1/public/signals` route selects `source_url` from the `signals` table, but that column does not exist in prod. Memory shows `signals.source_ids` is a `text[]` (referenced in `project_signal_detail_500.md` and `reference_api_slop_routes.md`). Either:
1. A migration that added `source_url` was never applied in prod, or
2. The route was recently updated to reference a column that is sourced via a join to `sources` and the join was dropped.

This is an extension of the schema-drift class of issues fixed Apr 15 (`project_signals_schema_drift.md` / `project_apr15_prod_fixes.md`) — a new column reference has slipped in.

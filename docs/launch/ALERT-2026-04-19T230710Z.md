# ALERT — Launch Day Health Check Failure

**Timestamp:** 2026-04-19T23:07:10Z (local 19:07)
**Severity:** HIGH — public API endpoint returning 400 on launch eve
**Status:** RECURRING — 5th consecutive failure (17:24, 17:35, 18:06, 18:36, 19:07)
**Hours-to-launch:** ~13h until 2026-04-20 launch day begins

## Failure

`GET https://api.world-pulse.io/api/v1/public/signals?limit=1` still returns **HTTP 400**.

Root cause (unchanged from ALERT-2026-04-19T223658Z.md): zod schema on the public
signals route expects `offset` as `z.string()`, but the querystring parser ahead of
it coerces the param to a number, so validation always fails:

```json
{"success":false,"code":"VALIDATION_ERROR","error":"offset: Expected string, received number"}
```

No patch has landed in the 31 minutes since the previous alert.

- `GET /api/v1/signals?limit=1` → 200 with `success:true`, signal data present
- `GET /api/v1/slop/stats` → 200; `total_signals_analyzed = 55,999` (up +86 since 18:36)

## Impact (escalating)

- 5 consecutive failures across ~103 minutes without a fix.
- Launch is tomorrow. Every hour this sits unpatched increases the chance the first
  HN/ProductHunt visitor hits a broken public API as their first impression.
- The documented, publicly-advertised route (`/api/v1/public/signals`) is the one
  listed in docs and likely embedded in any "Try the API" snippet on the marketing
  site. Third-party integrators copying from docs will fail on their first call.

## Recommended immediate actions (restated, urgency increased)

1. **Ship the fix tonight.** Patch `apps/api/src/routes/public/signals.ts` (or
   equivalent) so the `offset` schema tolerates numeric coercion — either
   `z.coerce.string()`, `z.union([z.string(), z.number()]).transform(String)`, or
   drop the string constraint entirely if offset is numeric.
2. After deploy, verify with:
   `curl -sS -w "\nHTTP:%{http_code}\n" "https://api.world-pulse.io/api/v1/public/signals?limit=1"`
   Expect `HTTP:200` and `data.items[…]`.
3. Add a pre-deploy smoke test (CI) that curls this exact documented URL so a future
   regression can't ship.

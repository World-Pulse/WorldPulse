  • nginx/conf.d/cloudflare.conf (NEW, 47L)

        2405:8100::/32, 2a06:98c0::/29, 2c0f:f248::/32
    EFFECT: $remote_addr and $binary_remote_addr now contain the real visitor
    IP → rate limiting zones work correctly from day 1 of Cloudflare routing.

  • apps/api/src/middleware/cloudflare.ts (NEW, 175L)
    - isCloudflareIp(ip): CIDR range membership check for all 15 CF IPv4 blocks
      using precomputed [start, end] integer pairs for O(n) lookup at startup
    - isValidCfRay(ray): regex guard (/^[0-9a-f]{16}-[A-Z]{2,5}$/i) — rejects
      spoofed or injection-laden CF-Ray values
    - isValidCfIp(ip): sanity guard (/^[\d.:a-fA-F]+$/, max 45 chars) — rejects
      "1.2.3.4; DROP TABLE users" style injection in CF-Connecting-IP
    - Fastify plugin (cloudflareMiddlewarePlugin):
        • Decorates req.cfClientIp — real client IP (CF-Connecting-IP or req.ip)
        • Decorates req.cfRay — Cloudflare trace ID (or undefined)
        • Decorates req.isBehindCloudflare — boolean detection flag
        • Response header X-CF-Ray: echoes CF-Ray for ops log correlation
        • Response header X-Real-Client-IP: resolved client IP for downstream
    - buildCfAwareKeyGenerator(req): rate-limit key function:
        Priority 1: user:${x-user-id header}  (authenticated user bucket)
        Priority 2: ip:${req.cfClientIp}       (real CF client IP)
        Priority 3: ip:${req.ip}               (fallback, trustProxy value)

  • apps/api/src/index.ts (UPDATED)
    - Import: cloudflareMiddlewarePlugin + buildCfAwareKeyGenerator
    - Register cloudflareMiddlewarePlugin BEFORE rateLimit plugin
      (so rate-limit keyGenerator can read req.cfClientIp)
    - rateLimit keyGenerator updated from inline lambda → buildCfAwareKeyGenerator

  • apps/api/src/__tests__/cloudflare-middleware.test.ts (NEW, ~250L, 20 tests)
    - isCloudflareIp: 5 groups (known CF IPs, non-CF IPs, loopback, IPv6, malformed)

    - buildCfAwareKeyGenerator: user-id priority, CF-IP path, fallback, undefined ip
    - Plugin integration (Fastify.inject): no-CF fallback, CF-Connecting-IP extraction,
